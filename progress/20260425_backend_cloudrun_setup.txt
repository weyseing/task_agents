What was fixed today
--------------------
1. LLM swap: removed Ollama entirely, moved to Anthropic.
     - backend/agent.py default model → anthropic/claude-haiku-4-5
     - Dropped OLLAMA_API_BASE, api_base branch, cli/ollama/
     - Removed extra_hosts: host.docker.internal from docker-compose.yml
     - Env: LITELLM_MODEL=anthropic/claude-haiku-4-5
            LITELLM_MODEL_DEEP=anthropic/claude-sonnet-4-6
            ANTHROPIC_API_KEY=<set>

2. Local dev fix: DATABASE_URL pointed at 'db:5432' but service is named
   'task_agents_db'. Docker DNS only resolves service names.
     - docker-compose.yml:29 → task_agents_db:5432

3. Cleanup: removed empty workspace/ dir and its mount.

4. Gmail local re-auth: OAuth token expired (Testing mode = 7-day lifetime).
   Deleted cli/gmail/token.json, re-ran python /cli/gmail/auth.py interactively.

5. Cloud Run public access — the big one.
   Blocker: --allow-unauthenticated silently no-ops when the deploy principal
   lacks run.services.setIamPolicy. roles/editor does NOT include it (carved
   out because binding allUsers is privileged). Deploy "Setting IAM Policy..."
   runs but the binding never persists → 403 at the edge.
   Fix: granted roles/run.admin to the deploy service account via a
   bootstrap script (cloud/setup/1_grant_gcloud_service_acc_permission.sh).
   After re-deploy, service IAM policy correctly shows:
     - allUsers → roles/run.invoker

6. Tooling added:
     - cloud/setup/1_grant_gcloud_service_acc_permission.sh
         One-time bootstrap to grant run.admin. Auto-uses a user account
         (via gcloud auth login) since the SA can't grant itself perms.
         Ends by reverting active account + revoking Owner creds.
     - cloud/cli/cloudrun/healthcheck_backend.sh
         Three-check smoke test (health / DB / chat), colors, --url,
         --message, --help. Auto-resolves URL via gcloud.
